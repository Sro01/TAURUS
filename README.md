# 기능 명세서

# 1. 예약 시스템 기본 규칙

### 1-1. 합주 팀 등록

- **팀 식별:** 팀명 / 비밀번호 (회원가입 없음)
- **등록 방법:**
    1. [신규 팀 등록] 메뉴를 통해 수동 등록
    2. [바로 예약] 진행 시 자동 등록 (최초 1회)

### 1-2. 타임 슬롯

- **규칙:** 매 시 정각 시작, 50분 진행 (10분 휴식)
- Ex)
    
    `13:00 ~ 13:50` / `14:00 ~ 14:50`
    

### 1-3. 메인 주차(Week) 기준

- **범위:** 월요일 09시 ~ 일요일 22시 50분
    - 즉, 각 요일별 타임 슬롯 시작 시간은 09:00 / 마지막 시간은 `22:00` (22:00~22:50)
- Ex)
    - 12월 5주차: 12/29(월) ~ 01/04(일)
    - 1월 1주차: 01/05(월) ~ 01/11(일)
- **메인 주차 전환 시점 (Reset):** 매주 일요일 18:00
    - 일요일 18시가 되면 메인 주차는 자동으로 **다음 주차**를 기준으로 변경됨
    - 초기화 때마다 “다음 주차(= 새 MainWeek)”의 각 슬롯별 신청 팀 수를 집계
        - 신청이 1팀이면 그 팀의 신청을 **`CONFIRMED`**로 승격
        - 2팀 이상이면 해당 슬롯의 신청을 전부 **`VOID`**로 처리(폭파)
    - Ex)
        
        02/01(일) 18시가 되면, 바로 예약 화면의 메인 주차로 2월 1주차(02/02~) 예약표가 뜸
        

### 1-4. 주의사항 & 최대 예약 슬롯

- **최대 예약 타임 슬롯 (팀당 제한)**
    - 각 팀의 매 주차별 예약 개수는 `CONFIRMED + PENDING ≤ MaxSlotsPerWeek` 를 항상 만족해야 함
    - **기본:** 주당 최대 2시간 (연속 불가, 1시간씩 분할 예약)
    - **공연 시즌:** 주당 최대 3시간 (**연속 2시간 가능**)
    - `MaxSlotsPerWeek`은 관리자 설정에 의해 변동 가능
- 슬롯 시작 시각이 현재 시각보다 **이전 또는 동일**이면 예약 불가
- 해당 규칙은 미리 예약, 바로 예약, 우선 예약 등 모든 예약 방식에 적용됨

### 1-5. 시나리오 예시 (Edge Case)

**상황:** 현재 시각 **02/01(일) 18:30분** 접속 시

- **[바로 예약] 페이지**
    1. **지난 주차 보기(**MainWeek의 이전 주차**):** 02/01(일) 19:00 ~ 22:00 (당일 저녁 예약 가능)
    2. **이번 주차 보기(**MainWeek**):** 02/02(월) 09:00 ~ 02/08(일) 22:00
- **[미리 예약] 페이지**
    - **다음 주차(**MainWeek의 다음 주차)**:** 02/09(월) 09:00 ~ 02/15(일) 22:00

# 2. 바로 예약 기능

### 2-1. 대상

- 이번 주차 (MainWeek) 월요일 09:00 ~ 일요일 22:00

### 2-2. 기능 상세

- **방식:** 선착순 (First Come First Served)
- 상태: **`CONFIRMED`**
    - 바로 예약으로 생성된 예약은 즉시 Confirmed됨
- **제약:**
    - 이미 예약된 시간대 선택 불가
    - '미리 예약'으로 확정된 시간대 선택 불가
- **표기:** 슬롯에 **예약된 팀명** 공개
- **팀 자동 등록:** 예약 성공 시 입력한 정보로 팀 등록 처리 (최초 1회)

### 2-3. 입력 정보

- 팀명 / 비밀번호

### 2-4. Edge Case

- 일요일 18시 이후에 일요일 당일 17시 ~ 22시 예약을 희망하는 경우
    - 일요일 오후 6시가 지나면 그 당일은 지난 주로 간주되기 때문에 지난 주차(MainWeek의 이전 주차) 화면으로 가서 예약 가능

---

# 3. 미리 예약 기능

### 3-1. 대상

- 다음 주차 (MainWeek의 다음 주차) 월요일 09:00 ~ 일요일 22:00

### 3-2. 기능 상세

- **방식:** 기간 내 자유 신청 (선착순 X)
- 상태: **`PENDING`**
    - 미리 예약으로 생성된 예약은 Pending 상태 
    ⇒ 주차가 전환될 때 조건에 따라 상태 변화
- **자격:** 기존에 등록된 팀만 가능 (팀 비밀번호 검증 필수)
    - *목적: 악의적인 예약 폭파 방지*
- **표기:** 각 타임슬롯별 예약 대기 중인 **팀 수(Count)** 표기 (팀명 비공개)
- 미리 예약 또한 주당 최대 예약 가능 슬롯 수와 동일

### 3-3. 성공 (확정) 조건

- **1팀 단독 신청 시:** 해당 주차 합주 예약 **확정**
- 상태: **`PENDING → CONFIRMED`**

### 3-4. 실패 (폭파) 조건

- **2팀 이상 중복 신청 시:** 해당 시간대 예약 **전부 무효화**
    - 바로 폭파되는 것은 아니며, 매주 일요일 18시 (초기화 시)에 최종 결정(1-4 참고)됨
    - *결과:* 해당 시간대는 '공석' 상태로 매주 일요일 18시 [바로 예약]에 오픈됨 (재예약 필요)
- 관리자 우선 예약은 즉시 **`CONFIRMED(ADMIN)`**로 생성되며, 그 슬롯은 모든 화면에서 선택 불가로 표시
- 상태: **`PENDING → VOID`**

### 3-5. 입력 정보

- 등록된 팀 리스트에서 선택 / 비밀번호 입력

# 4. 합주 팀 및 예약 관리

### 4-1. 합주 팀 리스트

- 현재 등록된 모든 팀 리스트 공개
- 팀명 클릭 시 [팀 관리 페이지]로 이동 (비밀번호 검증)

### 4-2. 합주 팀 관리 기능

- **예약 현황:** 내 팀의 주차별 예약 내역 확인 (상태값: 확정/대기)
- **예약 취소:** 확정된 예약 또는 대기 중인 예약 취소
- **정보 수정:** 팀명, 비밀번호 변경
- **팀 삭제:** 팀 데이터 영구 삭제

---

# 5. 관리자 기능

### 5-1. 접근 제어

- 관리자 마스터 패스워드로 검증

### 5-2. 주요 권한

- **우선 예약:** 모든 주차/시간대에 강제 예약 가능 (병아리밴드, 활동기수 우선권 배정용)
    - 관리자 우선 예약은 즉시 **`CONFIRMED_ADMIN`**로 생성되며, 그 슬롯은 모든 화면에서 선택 불가로 표시
- **강제 취소:** 규칙 위반(최대 예약 슬롯 초과 등) 적발 시 예약 강제 취소
- **팀 삭제:** 관리 차원에서 불필요한 팀 삭제 및 무분별하게 생성된 팀 관리
- **최대 슬롯 설정:** 팀 별로 최대로 예약 가능한 슬롯 수를 설정
(공연 시즌에 유동적으로 예약 시간을 조정하기 위함)
