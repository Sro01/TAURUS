# 1. 예약 시스템 기본 규칙

### 1-1. 합주 팀 등록

- **팀 식별:** 팀명 / 비밀번호 (회원가입 없음)
- **등록 방법:**
    1. [신규 팀 등록] 메뉴를 통해 수동 등록
    2. [바로 예약] 진행 시 자동 등록 (최초 1회)

### 1-2. 타임 슬롯

- **규칙:** 매 시 정각 시작, 50분 진행 (10분 휴식)
- Ex)
    
    `13:00 ~ 13:50` / `14:00 ~ 14:50`
    
- 각 요일 당 총 14개의 타임 슬롯 (09시 ~ 22시 매 시간 정각)
    - 즉, 각 요일별 타임 슬롯 시작 시간은 09:00 / 마지막 시간은 `22:00` (22:00~22:50)
- 모든 날짜 및 시간은 KST를 기준으로 함

### 1-3. 주차(Week) 기준

- **주차 계산 방식: 미국식 연도별 주차 체계 (일요일 시작)**
- 기본 원칙
    - **주 구성**: 일요일 00:00 ~ 토요일 23:59 (KST)
    - **요일 순서**: 일 월 화 수 목 금 토
    - **1주차 정의**: 해당 연도 1월 1일이 포함된 주
    - **주차 범위**: 연도별 1주차 ~ 52주차 또는 53주차
    - **유일 식별**: 각 주차는 시작일(일요일)로 고유하게 식별됨
- 주차 번호 부여 규칙
    1. 1월 1일이 포함된 주 = 그 해의 **1주차**
    2. 매주 일요일마다 주차 번호 증가 (2주차, 3주차, ...)
    3. 연말에는 52주차 또는 53주차로 종료
- 주차 표기 방식
    - **저장**: 연도별 연속 주차 번호 (weekNumber: 1~53)
    - **표기**: "N월 N주차" 형식으로 동적 계산
        - 계산 기준: 해당 주의 **수요일**이 속한 달로 판단
- 주차 계산 예시
    
    ```json
    2026년 1주차: 12/28(일) ~ 01/03(토)  ← 표기: "1월 1주차" (수요일=1/1)
    2026년 2주차: 01/04(일) ~ 01/10(토)  ← 표기: "1월 2주차"
    2026년 3주차: 01/11(일) ~ 01/17(토)  ← 표기: "1월 3주차"
    2026년 4주차: 01/18(일) ~ 01/24(토)  ← 표기: "1월 4주차"
    2026년 5주차: 01/25(일) ~ 01/31(토)  ← 표기: "1월 5주차"
    2026년 6주차: 02/01(일) ~ 02/07(토)  ← 표기: "2월 1주차" (수요일=2/4)
    ```
    

## 1-4. 메인 주차 전환 시점 (Reset)

- **전환 시각**: 매주 일요일 00:00 (KST)
- **전환 내용**:
    1. 이전 주차 → `CLOSED` 상태로 변경
    2. 현재 주차 → `OPEN` 상태로 변경
    3. 다음 주차 이후 → `UPCOMING` 상태 유지
    4. **PENDING 예약 집계 실행**:
        - 1팀만 신청한 슬롯 → `CONFIRMED` 승격
        - 2팀 이상 신청한 슬롯 → 모두 `VOID` 처리
- **주차 전환 시나리오 예시**
    
    ```json
    <2026년 2월 1일(일) 00:00 전환 시>
    
    전환 전:
      - 5주차 (01/25~01/31): OPEN  ← 바로 예약 가능
      - 6주차 (02/01~02/07): UPCOMING  ← 미리 예약만 가능
    
    전환 후:
      - 5주차 (01/25~01/31): CLOSED  ← 과거 주차
      - 6주차 (02/01~02/07): OPEN  ← 바로 예약 화면에 표시
      - 7주차 (02/08~02/14): UPCOMING  ← 미리 예약 화면에 표시
      
    바로 예약 화면:
      → "2월 1주차 (02/01~02/07)" 예약표가 표시됨
      → 02/01(일) 09:00부터 예약 가능
    ```
    
- 추가 예시: 연말/연초 경계
    
    ```json
    **< 2025년 12월 ~ 2026년 1월 >**
    
    2025년 52주차: 12/21(일) ~ 12/27(토)  ← 표기: "12월 4주차"
    2026년  1주차: 12/28(일) ~ 01/03(토)  ← 표기: "1월 1주차" (2026년 1/1 포함)
    2026년  2주차: 01/04(일) ~ 01/10(토)  ← 표기: "1월 2주차"
    
    < 2025년 12월 28일(일) 00:00 전환 시 >
    - 2025년 52주차 → CLOSED
    - 2026년 1주차 → OPEN (새해 첫 주!)
    - 사용자 화면: "1월 1주차" 표시
    ```
    

### 1-5. 주의사항 & 최대 예약 슬롯

- **최대 예약 타임 슬롯 (팀당 제한)**
    - 각 팀의 매 주차별 예약 개수는 `CONFIRMED + PENDING ≤ MaxSlotsPerWeek` 를 항상 만족해야 함
    - **기본:** 주당 최대 2시간 (연속 불가, 1시간씩 분할 예약)
    - **공연 시즌:** 주당 최대 3시간 (**연속 2시간 가능**)
    - `MaxSlotsPerWeek`은 관리자 설정에 의해 변동 가능
- 슬롯 시작 시각이 현재 시각보다 **이전 또는 동일**이면 예약 불가
- **팀 자동 등록:** 예약 성공 시 입력한 정보로 팀 등록 처리 (최초 1회)
- 해당 규칙은 미리 예약, 바로 예약, 우선 예약 등 모든 예약 방식에 적용됨
- 주차 겹침 문제
    - 연도 전환이 이루어질 때, 올해의 마지막 주차와 내년의 첫 번째 주차의 날짜 범위가 겹치는 문제 발생
    
    **해결**
    
    - 선제적 2개년 데이터 관리
        
        : 초기화 시 현재 연도뿐만 아니라 '내년' 데이터까지 미리 생성하도록 하여, 연말에 다음 주차를 조회할 때 발생할 수 있는 런타임 에러를 방지

# 2. 바로 예약 기능

### 2-1. 대상

- 이번 주차 (ThisWeek) 일요일 ~ 월요일 (09:00 - 09:50 ~ 22:00 - 22:50 (KST))

### 2-2. 기능 상세

- **방식:** 선착순 (First Come First Served)
- 상태: **`CONFIRMED`**
    - 바로 예약으로 생성된 예약은 즉시 Confirmed됨
- **제약:**
    - 이미 예약된 시간대 선택 불가
    - '미리 예약'으로 확정된 시간대 선택 불가
- **표기:** 슬롯에 **예약된 팀명** 공개

### 2-3. 입력 정보

- 팀명 / 비밀번호 / 설명

---

# 3. 미리 예약 기능

### 3-1. 대상

- 다음 주차 (NextWeek) 일요일 ~ 월요일 (09:00 - 09:50 ~ 22:00 - 22:50 (KST))

### 3-2. 기능 상세

- **방식:** 기간 내 자유 신청 (선착순 X)
- 상태: **`PENDING`**
    - 미리 예약으로 생성된 예약은 Pending 상태 
    ⇒ 주차가 전환될 때 조건에 따라 상태 변화
- **표기:** 각 타임슬롯별 예약 대기 중인 **팀 수(Count)** 표기 (팀명 비공개)
- 미리 예약 또한 주당 최대 예약 가능 슬롯 수와 동일

### 3-3. 성공 (확정) 조건

- **1팀 단독 신청 시:** 해당 주차 합주 예약 **확정**
- 상태: **`PENDING → CONFIRMED`**

### 3-4. 실패 (폭파) 조건

- **2팀 이상 중복 신청 시:** 해당 시간대 예약 **전부 무효화**
    - 바로 폭파되는 것은 아니며, 매주 일요일 18시 (초기화 시)에 최종 결정(1-4 참고)됨
    - *결과:* 해당 시간대는 '공석' 상태로 매주 일요일 18시 [바로 예약]에 오픈됨 (재예약 필요)
- 관리자 우선 예약은 즉시 **`CONFIRMED(ADMIN)`**로 생성되며, 그 슬롯은 모든 화면에서 선택 불가로 표시
- 상태: **`PENDING → VOID`**

### 3-5. 입력 정보

- 팀명 / 비밀번호 / 설명

# 4. 합주 팀 및 예약 관리

### 4-1. 합주 팀 리스트

- ~~현재 등록된 모든 팀 리스트 공개~~
- ~~팀명 클릭 시 [팀 관리 페이지]로 이동 (비밀번호 검증)~~
- 모든 팀 리스트 X → 메뉴바에서 팀 관리 페이지 클릭

### 4-2. 합주 팀 관리 기능

- **예약 현황:** 내 팀의 주차별 예약 내역 확인 (상태값: 확정/대기/취소)
- **예약 취소:** 확정된 예약 또는 대기 중인 예약 취소
- **정보 수정:** 팀명, 비밀번호 변경
- **팀 삭제:** 팀 데이터 영구 삭제

# 5. 관리자 기능

### 5-1. 접근 제어

- 관리자 마스터 패스워드로 검증

### 5-2. 주요 권한

- **우선 예약:** 모든 주차/시간대에 강제 예약 가능 (세션 레슨, 병아리밴드, 활동기수 우선권 배정용)
    - 관리자 우선 예약은 즉시 **`CONFIRMED_ADMIN`**로 생성되며, 그 슬롯은 모든 화면에서 선택 불가로 표시
- **강제 취소:** 모든 예약을 강제 취소 가능 (관리자 권한 예약 및 기타 무분별한 예약을 관리)
- **팀 관리:** 모든 팀 리스트 조회 및 삭제 (무분별하게 생성된 팀 관리)
- **최대 슬롯 설정:** 팀 별로 최대로 예약 가능한 슬롯 수를 설정
(공연 시즌에 유동적으로 예약 시간을 조정하기 위함)
